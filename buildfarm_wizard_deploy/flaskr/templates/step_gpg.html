GPG Key

The GPG key is used to sign the packages generated by the buildfarm and uploaded to the repo.

A GPG key has an identifier, a public, and a private part.

If you already have a GPG key, you can enter the information requested below. Otherwise you can generate a gpg key by running:

gpg --gen-key

It is important that the gpg key's passphrase must be empty.

After running the key generation command and answering the questions, a message like the following one will be shown:

gpg: key 8E19F126 marked as ultimately trusted

The 8E19F126 string is the identification string for the signing key.

Tip

Use gpg --export-secret-keys -a [key_identifier] to retrieve the private key, and gpg --export -a [key_identifier] to retrieve the public key.
